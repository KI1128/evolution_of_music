# The River of Music

## 概要
本作は、1400年代のクラシック音楽から現代のポップスに至るまでの音楽ジャンルの系譜と歴史的マイルストーン（名盤・重要曲）を、ストリームグラフ（Sankey風）として可視化するジェネレーティブ・アート・ジェネレーターです。
データに基づく正確な歴史の繋がりを保ちつつ、鑑賞に堪えうるポスターアートとしての美しさを両立するように設計されています。

## コア・コンセプト
1. **途切れない大河**: どのジャンルも「死ぬ」ことはなく、現在（実行年の現在）まで帯として伸び続ける。
2. **歴史の脈動**: 歴史的マイルストーンが存在する年代において、ジャンルの帯は熱を帯びて太く膨張し、その後緩やかに減衰する。
3. **DNAの継承**: 親ジャンルから子ジャンルへの派生は、親の「色」を引き継ぎながら滑らかなS字カーブで合流する。

## ファイル構成
保守性と拡張性を高めるため、役割ごとに4つのファイルに分割されています。

* `data.py`: **[重要]** ジャンル定義、親子関係、マイルストーン、表示年代など、すべての「データ」を一元管理するファイル。後任者が日常的に触るのはこのファイルのみです。
* `layout.py`: 時間軸の非線形変換（古い時代を圧縮し、近代を広げる）や、NetworkXを用いたノードの初期Y座標計算など、数学的なロジックを担当。
* `render.py`: Matplotlibを用いた描画エンジン。ストリームの動的膨張、S字カーブの補間計算、テキストラベルの衝突回避（物理演算）など、ビジュアル生成の心臓部。
* `main.py`: 実行用のエントリーポイント。

## 環境構築
動作には以下のPythonライブラリが必要です。

```bash
pip install matplotlib networkx numpy

```

## 実行方法

```bash
python main.py

```

実行すると計算が行われ、同階層に超高解像度のアート画像（`music_tree_dynamic.png`）が出力されます。

## データの追加・更新ルール (`data.py`)

### 1. ジャンルの追加 (`music_genres` 辞書)

新しいジャンルを追加する場合は、以下のフォーマットで追記してください。

```python
"Genre Name": {
    "year": 1990, # 誕生年
    "parents": ["Parent1", "Parent2"], # 派生元のジャンルリスト
    "volume": 80, # 帯の基本となる太さ（影響力）
    "color": "#HEXCODE" # ルーツに合わせたテーマカラー
}

```

### 2. マイルストーンの追加 (`milestones` リスト)

名盤や歴史的な転換点を追加すると、自動的にその年代の帯が膨らみ、ラベルが描画されます。

* 選定基準: 「ジャンルの誕生」「そのジャンルの絶対的代名詞」「価値観を変えた突然変異」を優先してください。

```python
{
    "year": 1991, # 発表年
    "genre": "Alternative Rock", # 属するジャンル名（music_genresと完全に一致させること）
    "artist": "Nirvana",
    "work": "Nevermind"
}

```

### 3. 表示年代の調整 (`DISPLAY_YEARS`)

出力画像の下部に表示されるX軸のラベルです。データが増えて時代が偏ってきた場合は、このリストの数値を変更して見栄えを調整してください。

## 描画エンジンの仕様に関する注意点 (`render.py`)

* **テキストの衝突回避**: `render.py` 内で、ラベル同士が重ならないようにY座標を微調整する50回の反復シミュレーションを行っています。マイルストーンを極端に密集させすぎると計算が追いつかなくなる場合があるため、重要作品の厳選を推奨します。
* **キャンバスサイズの自動拡張**: ジャンル数が増えると、自動的にキャンバスの縦幅が伸びる仕様になっています。
